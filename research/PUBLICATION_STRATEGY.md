## 2. Publishable Core Innovations

### I. The "Oracle-Based" Verification Framework (Methodological)
This is a novel software engineering pattern for migrating complex symbolic systems. It addresses the "identity crisis" and "simplification gap" inherent in porting symbolic logic between different Computer Algebra Systems (CAS).
- **The 3-Tier Verification Strategy:**
    - **Tier 1 (AST-Based Normalization):** A formal tree-based canonicalization engine that out-performs regex-based approaches for nested tensor expressions.
    - **Tier 2 (Symbolic Oracle):** Using a Dockerized Wolfram Engine as a "living reference" for symbolic equivalence.
    - **Tier 3 (Numeric Tensor Realization):** Verifying symbolic identities (e.g., the First Bianchi Identity) via random, well-conditioned numeric sampling of tensor components—a fallback for cases where symbolic simplification fails.

### II. XCore.jl: The First Verified Open-Source xAct Port (Implementation)
The implementation of `XCore.jl` (and its Python counterpart) is highly impactful.
- **Democratization of GR Research:** It removes the dependency on expensive, proprietary Mathematica licenses for high-end tensor calculus.
- **Verification Pedigree:** Unlike other ports, `XCore.jl` is formally verified against the original `xAct` implementation using the Oracle framework.
